# Mobile-Architect-Programming
snhu mobile architect &amp; programming
Developing this weight tracker app gave me the chance to build a complete Android application from start to finish. The goal was to create an app where users could register an account, log in, record their weight, and easily view previous entries. Instead of making something overly complicated, I wanted the app to focus on one main purpose and do it well. It was designed for users who want a simple way to keep track of their progress without needing a full fitness app with a lot of extra features.

To make the app easy to use, I included separate screens for logging in, creating a new account, viewing the dashboard, adding weight entries, and managing notifications. I kept the interface simple by using consistent colors, readable text, and buttons that were easy to find. The dashboard was designed to show the user's current weight, goal weight, and previous entries in one place so they wouldn't have to search through different screens. I think this layout makes the app straightforward and user-friendly while still looking clean and organized.

While building the app, I focused on completing one feature at a time before moving on to the next. Breaking the project into smaller pieces made it much easier to stay organized and troubleshoot problems as they came up. I also spent a lot of time testing the app in the Android emulator after almost every change. That process helped me catch bugs like crashes, database errors, and layout problems before they became even bigger issues. By the end of the project, I had become much more comfortable reading error messages and using Logcat to figure out what was causing problems.

The biggest challenge was connecting everything together so the app worked as one complete system. There were several times when the database wasn't saving information correctly or the dashboard wasn't displaying data the way I expected. Working through those issues taught me a lot about debugging and how important it is to test each feature as you build it. Looking back, I'm most proud of getting the login system, SQLite database, and dashboard working together. Watching users log in, save new weight entries, and immediately see their information update made all the debugging worth it. This project really showed me how much my Android development skills have improved since the beginning of the course.
